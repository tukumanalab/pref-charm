# UI デザインガイドライン

AIっぽさのない、実運用に耐えるUIを設計するためのガイドライン。

## 1. 情報設計と視線誘導

- 画面内で何が主役かをひと目で分かるようにする
- 情報の優先順位を明確にし、「最初に見る → 次に比較する → 最後に行動する」という視線の流れを設計する
- よくあるUIパターンを機械的に並べるのではなく、この業務・体験に合った情報設計から考える
- ただ整っているだけの平均点デザインではなく、優先順位にメリハリをつける

## 2. 色の役割設計

色には明確な役割を持たせ、同じ色に複数の意味を持たせない。

| 役割 | 用途 | 注意点 |
|------|------|--------|
| ブランド色 | アプリの識別、ヘッダー等 | 多用しすぎない |
| 操作色 | ボタン、リンク等のインタラクション | 操作可能であることが一目で分かること |
| 状態色 | 成功・進行中・完了等の状態表示 | 操作色と混同しない |
| 注意色 | エラー・警告・削除等の注意喚起 | 赤系は注意色に限定する |

- 見た目の華やかさではなく、意味が伝わる色使いにする
- 装飾目的だけの色は使わない

## 3. 実運用への耐性

理想的なモックではなく、実際に使われるプロダクトとして設計する。

以下のケースで破綻しないことを確認する:
- 長いテキスト（ユーザー名、クエストタイトル等）
- データの欠損（画像なし、説明文なし等）
- 異常値（極端に大きい数値等）
- ゼロ件（検索結果なし、一覧が空等）
- 重複データ

## 4. 引き算のデザイン

- 情報や装飾を盛りすぎず、本当に必要な要素に絞る
- なくても困らない装飾や要素は削ぎ落とす
- 「足してから引く」ではなく「必要なものだけ置く」
- 余白は情報整理の手段として積極的に使う

## 5. やってはいけないこと

- 角丸カード＋グラデーション＋アイコンを並べただけの汎用的なレイアウト
- 意味のないイラストやアイコンの装飾
- 全要素を均等に目立たせる（＝何も目立たない）デザイン
- 機能ごとに異なるデザイン言語を使う
- 状態を色だけで表現する（テキストやアイコンも併用する）
